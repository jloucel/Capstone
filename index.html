Capstone Project - Broken Link Finder

The goal of this project was to consider how software development skills could be utilized in a QA role. With this goal in mind I began thinking of standalone test utilities that could be used in any environment. This project satisfied both of these objectives, as well as reinforcing key concepts taught in the Computer Science program at Westminster College.

Selected Course Work

  • C Virtual Memory Manager

    This project simulates virtual memory location access, translates virtual addresses into physical memory addresses, and then outputs the value stored in the physical location. Utilizing virtual memory allows operating systems to allocate exponentially more memory to processes and remain performant with limited physical memory resources.

    Learning Goals satisfied by this project:

    • Demonstrate proficiency in developing software in more than one programming language and programming paradigm.
    • Demonstrate how computer hardware executes a program.

  • Scheme Interpreter in Scala

    This project implements a Scheme language interpreter in Scala. The development of the final interpreter was accomplished by incrementally. By adding more complex logic features such as conditional control structures, lambdas, and environments the interpreter gained enough functionality to support parsing a Scheme language program. It could then successfully execute a Scala program to produce the expected output of the parsed Scheme program.

    • Critical, analytical and integrative thinking.

  • Java Greedy vs Dynamic Algorithms

    This project contrasts the finding of an optimal solution to a path problem between greedy and dynamic algorithms. To illustrate this behavior a randomly generated board of number tiles is traversed using the desired algorithm, with a score returned after traversal. The user can then observe if the greedy or dynamic algorithm discovered the optimal path. The difference between the two algorithms is in how the next solution is derived, the greedy approach yields the best solution at a given step where the dynamic approach tries all solutions and can then choose the optimal solution at each step.

  • Sierpinski's Triangle in Elm

    Elm is a purely functional programming language that compiles to HTML, CSS, and Javascript. The goal of this project was to learn how to create an interactive program using Elm. I choose to implement Sierpinski's triangle and provide the user with a slider to control the number of iterations generated.

  • Unit Testing in Python

    I am including this program as an example of utilizing unit testing in general, as well as to suggest that testing can never be exhaustive. This is a simple program to generate a matrix of coprime pairs. Where a pair exists an '*' is stored in the matrix (4,9) for example. I feel at 10 tests it has been sufficiently, possibly over, tested, but not exhaustively tested.
